Hướng dẫn kết nối hai laptop qua mạng Wi-Fi
Dưới đây là các bước chi tiết để thiết lập kết nối giữa máy của bạn (server) và máy của bạn bạn (client) qua mạng Wi-Fi, với mã nguồn đã được điều chỉnh.

Phần 1: Kiểm tra địa chỉ IP của máy server
Trên máy của bạn (server), kiểm tra địa chỉ IP trên mạng Wi-Fi:
Windows:
Mở Command Prompt (Win + R > cmd).
Gõ ipconfig.
Tìm "IPv4 Address" trong "Wireless LAN adapter Wi-Fi", ví dụ: 192.168.1.100.

macOS:
Mở Terminal.
Gõ ifconfig | grep inet.
Tìm "inet" của Wi-Fi (thường là en0), ví dụ: 192.168.1.100.

Linux:
Mở Terminal.
Gõ ip addr show | grep inet.
Tìm "inet" của Wi-Fi (thường là wlan0), ví dụ: 192.168.1.100.
Ghi lại IP (ví dụ: 192.168.1.100) và thông báo cho bạn bạn.

Phần 2: Cấu hình mã nguồn client
Trên máy của bạn bạn (client):
Mở chatClient.java bằng trình soạn thảo (Notepad, VS Code, v.v.).
Tìm dòng:
String serverAddress = "localhost";

Thay bằng IP của máy bạn (server), ví dụ:
String serverAddress = "192.168.1.100";

Giữ nguyên port:
int serverPort = 2025;

Lưu file.

Phần 3: Mở port trên máy server
Máy của bạn (server) cần mở port 2025:
Windows:
Mở "Windows Defender Firewall with Advanced Security" (Win + S > "firewall").
Chọn "Inbound Rules" > "New Rule".
Chọn "Port" > "TCP" > nhập 2025 > "Allow the connection" > áp dụng tất cả > đặt tên "Allow Port 2025" > Finish.
macOS:
Nếu firewall bật: System Settings > Network > Firewall > tắt hoặc thêm java vào danh sách cho phép.
Linux:
Gõ sudo ufw allow 2025/tcp > kiểm tra sudo ufw status.
Kiểm tra: Trên máy bạn, gõ nc -l 2025 (cần cài netcat). Nếu không lỗi, port đã mở.

Phần 4: Cấu hình tường lửa
Máy của bạn (server) cần cho phép kết nối qua port 2025:
Windows: Nếu đã mở port, không cần thêm. Tắt tạm: "Windows Defender Firewall" > "Turn off" cho "Private".
macOS: Tắt firewall hoặc thêm java (như trên).
Linux: Nếu dùng ufw, đã mở ở bước trên. Tắt tạm: sudo ufw disable.
Máy của bạn bạn (client) không cần cấu hình tường lửa.

Phần 5: Chạy chương trình
Trên máy của bạn (server):
Đảm bảo JDK đã cài (java -version và javac -version).
Mở Terminal/Command Prompt, vào thư mục mã nguồn:
cd path/to/your/project

Biên dịch:
javac chatbot/*.java

Chạy:
java chatbot.chatServer
Thấy: Server dang chay va cho ket noi....

Trên máy của bạn bạn (client):
Đảm bảo JDK đã cài và serverAddress đã cập nhật.
Mở Terminal/Command Prompt, vào thư mục mã nguồn:
cd path/to/your/project

Biên dịch:
javac chatbot/*.java

Chạy:
java chatbot.chatClient

Phần 6: Kiểm tra kết nối
Dấu hiệu thành công:
Trên máy bạn (server):
Server dang chay va cho ket noi...
Client DESKTOP-ABC123 da ket noi.
Client bao ket noi tu server: DESKTOP-XYZ789

Trên máy bạn bạn (client):
Đã kết nối với server: DESKTOP-XYZ789
=======================================
|    CHUONG TRINH BAI TAP JAVA        |
=======================================
1. Kiem tra so
2. Tong va tich cac chu so
3. Uoc chung lon nhat va boi chung nho nhat
4. Dao chuoi
5. Xu ly chuoi ky tu
6. Tan so xuat hien ky tu trong chuoi
0. Thoat
=======================================
[Server] Chon bai tap (1-6, 0 de thoat):
Bạn bạn có thể nhập số (ví dụ: 1) và tương tác.

Nếu thất bại:
Kiểm tra IP: Đảm bảo IP trong chatClient.java khớp với IP hiện tại của máy bạn.
Kiểm tra server: Đảm bảo server chạy trước client.
Kiểm tra mạng: Từ máy bạn bạn, ping máy bạn (ping 192.168.1.100). Nếu không phản hồi, kiểm tra Wi-Fi.
Kiểm tra port/tường lửa: Đảm bảo port 2025 mở và không bị chặn trên máy bạn.

Lưu ý
Nếu hostname không phân giải được (DNS yếu), bạn sẽ thấy IP thay vì tên máy. Để khắc phục, cấu hình file hosts hoặc dùng IP tĩnh.
Đảm bảo cả hai máy cài JDK (ít nhất Java 8) và có đầy đủ file phụ thuộc (So.java, Chuoi.java).
